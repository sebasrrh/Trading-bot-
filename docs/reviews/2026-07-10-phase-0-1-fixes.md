# Review & fixes — Phase 0/1 implementation (2026-07-10)

Audience: the agent (or human) continuing implementation. This documents what
was changed in the review-fix commit, **why**, and what is still open. Read
this before writing more Phase 1/2 code — several of these were process
failures, not typos, and they will recur unless the workflow changes.

## Process rules going forward (the actual lesson)

1. **Run what you commit.** Every one of the five blockers below would have
   been caught by: fresh clone → `pnpm install` → `pnpm typecheck && pnpm test
   && pnpm lint && pnpm build` → `pnpm dev` → look at the page. Do this before
   every commit. CI now enforces the first four.
2. **This project builds on Linux CI.** No Windows-only commands in scripts
   (`copy /Y` broke the ui build). Use POSIX or Node APIs.
3. **Don't commit tooling artifacts or scratch files.** `.swarm/` telemetry,
   `bar-fix.txt`, `bar-series-temp.txt` were committed. `.swarm/` is now
   gitignored; scratch files don't belong in PRs.
4. **Silent failure is a bug by policy** (docs/04): a provider adapter that
   catches an error and returns `[]` turns outages into "the market has no
   data". Errors must propagate to logs and to the response.

## What was changed and why

### 1. Design system was never wired up
- `apps/web/src/main.tsx` now imports `@tradeboard/ui/tokens.css`. The token
  sheet (with its reset + body styles) existed but nothing imported it, so the
  app rendered as an unstyled white serif page.
- `packages/ui/package.json` exposes `"./tokens.css": "./src/tokens.css"` and
  the build script is plain `tsc` (was `tsc && copy /Y src\tokens.css …`,
  which fails on Linux).
- Nav rail emoji (📊🧪📈💰) replaced with `lucide-react` icons per docs/02 §6
  (`lucide-react` added to `apps/web` deps — it was only a ui dep before, and
  pnpm strict node_modules forbids undeclared imports).

### 2. Preset workspaces rendered "No widgets yet"
- Presets defined grid `layouts` but left `widgets: {}` — the grid renders
  from `workspace.widgets`, so every preset was empty. A `preset()` helper in
  `workspace-store.ts` now builds layout entries and widget instances with
  **shared, stable ids** (`candles-1`), defaulting every widget to channel A.
- `addWidget` never wrote a layout entry, so added widgets depended on
  react-grid-layout auto-placement. It now appends a layout item at the bottom
  using the manifest's `defaultSize` (callers pass it; the store must not
  import the registry — that would be an import cycle).
- `dashboard-grid.tsx`: layout is stored-layout ∪ synthesized-entries for
  instances the layout doesn't know (heals workspaces saved by the old code),
  minus entries whose instance is gone.
- The drag handle: `draggableHandle=".widget-drag-handle"` was configured but
  no element had that class — dragging was impossible. The widget header now
  carries it.
- Top-bar `+ Widget` opened nothing useful (hardcoded `addWidget('candles')`);
  it now opens the command palette's Add Widget tab.

### 3. Fresh clone was red (typecheck, tests, CI)
- All `packages/*` now resolve from **source**: `main`/`types`/`exports` point
  at `./src/index.ts`. With `moduleResolution: "bundler"`, tsc type-checks the
  .ts directly and Vite/tsx/Vitest compile it — no build step needed to
  consume a workspace package, no stale-dist bugs. (`dist/` output via
  `pnpm build` still works and is what a future publish/docker build uses.)
- `packages/strategies` used `zod` without declaring it → added to its deps.
- ESLint was completely unconfigured for TypeScript (espree choked with 49
  parse errors — lint had never passed). Now `typescript-eslint` flat config;
  `lint` script is `eslint .` (ESLint 9 dropped `--ext`).
  `@typescript-eslint/no-explicit-any` is **off for now** because widget/config
  boundaries are `any`-typed — see Open items #2.
- CI (`.github/workflows/ci.yml`) additionally runs `pnpm build` so a broken
  Vite build can't merge.

### 4. Data providers failed silently (and two were broken anyway)
- **Stooq symbols need the market suffix**: `spy.us`, not `spy`. Bare symbols
  return an empty CSV — bars were empty even when the network was fine.
- **Yahoo adjusted-price math was wrong**: `o * adj / (c / o)` (≈ o²·adj/c).
  Correct factor is `f = adjclose / close`, applied to o/h/l/c. Backtests on
  the old formula would have been garbage.
- Yahoo requests now send a browser `User-Agent` (default fetch UA gets
  403/429 from Yahoo).
- Adapters **throw** on HTTP failure instead of returning `[]`. The router
  logs each provider failure (`console.warn('[provider] …')`), collects them
  as `warnings`, and distinguishes three outcomes:
  - a provider succeeded → `{ source: providerId, data, warnings }`
  - all capable providers **errored** → `AllProvidersFailedError` → routes
    return **502** `{ error, warnings }`
  - providers responded but had nothing → `source: null`, empty data, 200.
- `/api/bars` served purely from SQLite now reports `source: 'cache'` (was
  hardcoded `'stooq'`). All responses include `warnings: string[]`.

### 5. Repo hygiene
- Deleted `.swarm/` (agent telemetry/session logs) and
  `packages/bar-fix.txt`, `packages/bar-series-temp.txt`; `.swarm/` gitignored.

## Open items, in priority order

1. **Verify live data end-to-end** (see above), including the quote strip and
   watchlist deltas (quotes come only from Yahoo — Stooq has no quote
   capability).
2. **Replace `any` config plumbing with zod schemas** per docs/03 §3
   (`configSchema` on the widget manifest, parse on load, auto-form later).
   Then re-enable `@typescript-eslint/no-explicit-any`.
3. **Top-bar data-source badge is hardcoded "Stooq"** — wire it to the
   `source`/`warnings` of the most recent bars/quotes response and to
   `/api/health`, amber (`--warn`) when `warnings` is non-empty or source is
   `cache`/fallback (docs/03 §1, docs/04 §1).
4. **Grid responsiveness**: width is hardcoded `1200` — use
   `WidthProvider(Responsive)` from react-grid-layout with the lg/md
   breakpoints from docs/03 §2. Also pass real `wPx/hPx` into `WidgetProps.size`
   (currently `{0,0}`).
5. **`/api/health` provider booleans are hardcoded** — derive from configured
   env keys + a cheap reachability probe, and include last-error info.
6. **Bars cache coverage check** (`meta.first_t <= from`) never matches the
   default `from=0` → cache is effectively bypassed for open-ended requests.
   Implement range subtraction per docs/04 §4, and decide sql.js vs
   better-sqlite3 (spec says better-sqlite3; sql.js is in-memory WASM with a
   manual file flush — if it stays, document why and make persistence robust).
7. **Workspace features from docs/03 §6**: export/import JSON, share string,
   reset-to-preset, md breakpoint layout. `Workspace.version` exists — bump to
   2 when the shape changes and write the migration.
8. Missing Phase 1 acceptance bits: IndexedDB persistence for bar queries
   (docs/04 §5), settings view, `X-Data-Source` header equivalent (we chose
   body fields — update docs/04 §1 wording or add the header).
